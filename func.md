下面按**学生、教师、教务管理员**三类核心角色，对系统功能进行分析和归纳。


# 一、三类角色总体定位

| 角色    | 核心定位           | 在系统中的主要作用                       |
| ----- | -------------- | ------------------------------- |
| 学生    | 成长数据的查看者与主动提交者 | 查看自己的成长画像、提交课堂表现和荣誉、查看审核结果      |
| 教师    | 成长数据的主要记录者与审核者 | 评价学生表现、审核学生提交内容、导入成绩、录入荣誉、查看画像  |
| 教务管理员 | 系统规则与基础数据的管理者  | 维护评价规则、成绩规则、五育权重、原因卡片、荣誉类型、基础数据 |

系统的核心逻辑是：

```text
学生提交 / 教师记录
→ 教师审核
→ 数据进入成长档案
→ 系统生成五育画像
→ 学生、教师、家长查看成长结果
→ 反向指导教学和学生发展
```


# 二、学生角色功能分析

学生端的重点不是管理数据，而是**查看自己的成长情况，并主动补充成长表现**。

## 1. 学生首页

学生登录后，应看到自己的成长概览，包括：

1. 我的成长指数
2. 我的五育雷达图
3. 我的本学期表现
4. 我的成长趋势
5. 我的荣誉记录
6. 我的待审核提交
7. 我要提交表现
8. 我要提交荣誉

学生首页应突出“我现在怎么样”“我有哪些优势”“我还需要改进什么”。


## 2. 查看个人成长画像

学生可以查看自己的完整成长档案，包括：

| 功能    | 内容                    |
| ----- | --------------------- |
| 五育雷达图 | 德育、智育、体育、美育、劳育当前发展情况  |
| 成长指数  | 当前综合成长指数、本学期成长指数      |
| 成长趋势  | 从一年级至当前年级的长期变化        |
| 学科表现  | 语文、数学、英语、科学成绩趋势       |
| 课堂表现  | 教师评价和已通过的自评记录         |
| 重点荣誉  | 获得的主要竞赛、活动、评比荣誉       |
| 优势标签  | 如数学思维突出、礼仪表现优秀、艺术表现活跃 |
| 待提升标签 | 如课堂专注度需提升、运动参与不足      |

学生查看画像的目的，是帮助学生认识自己的成长轨迹，而不是单纯查看分数排名。


## 3. 提交课堂表现

学生可以主动提交自己的课堂或活动表现，但提交后**不能直接计入画像**，必须经过教师审核。

操作流程：

```text
选择评价入口
→ 选择评价维度
→ 选择分值
→ 选择原因卡片
→ 填写补充说明
→ 上传证明材料，可选
→ 提交教师审核
```

学生可以选择的评价入口包括：

1. 科学
2. 英语
3. 数学
4. 语文
5. 礼仪
6. 运动
7. 艺术
8. 劳动

评价维度包括：

1. 学习态度
2. 课堂常规
3. 学业水平
4. 竞赛活动


## 4. 提交荣誉记录

学生可以提交自己获得的竞赛、活动、评比、荣誉信息。

提交内容包括：

1. 活动名称
2. 活动类型
3. 所属五育分类
4. 荣誉级别
5. 奖项类型
6. 证书图片
7. 获奖时间

提交后状态为：

```text
待审核
```

教师审核通过后，荣誉才会进入学生成长档案，并参与画像统计。


## 5. 查看审核状态

学生可以查看自己提交内容的处理进度。

| 状态  | 含义             |
| --- | -------------- |
| 待审核 | 已提交，等待教师处理     |
| 已通过 | 教师审核通过，已计入成长档案 |
| 已退回 | 内容需要修改后重新提交    |
| 未通过 | 内容不符合要求，不计入画像  |

如果被退回，系统应显示教师填写的退回原因，例如：

```text
请补充活动证明材料。
分值选择偏高，建议调整后重新提交。
该记录与之前提交内容重复。
```


## 6. 学生端功能边界

学生可以：

1. 查看自己的数据。
2. 提交自己的表现。
3. 提交自己的荣誉。
4. 查看审核结果。

学生不可以：

1. 修改教师评价。
2. 直接让自评进入画像。
3. 查看其他学生完整画像。
4. 修改成绩、规则、权重、原因卡片。


# 三、教师角色功能分析

教师是系统中最重要的业务使用者，承担**评价、审核、成绩导入、荣誉录入、画像查看**等工作。

## 1. 教师首页

教师登录后应看到一个工作台，主要包括：

1. 今日课程
2. 快速评价入口
3. 待审核学生自评
4. 待审核学生荣誉
5. 最近评价记录
6. 任教班级学生列表
7. 成绩导入入口
8. 学生画像入口
9. 班级评价概览

教师首页的重点是提高日常评价效率，让教师快速完成记录和审核。


## 2. 课堂动态评价

教师可以对学生进行课堂动态评价。

支持两种方式：

| 方式     | 说明                |
| ------ | ----------------- |
| 单个学生评价 | 对某个学生进行个别评价       |
| 批量评价   | 对多个学生、小组或全班进行相同评价 |

教师评价流程：

```text
选择班级
→ 选择学生
→ 选择评价入口
→ 选择评价维度
→ 选择分值
→ 选择原因卡片
→ 填写补充说明
→ 提交评价
→ 直接进入学生成长档案
```

教师直接评价默认有效，不需要二次审核。


## 3. 批量评价

教师可以多选学生后批量评价，适用于：

1. 小组合作表现优秀。
2. 多名学生共同参加活动。
3. 全班完成某项任务。
4. 多名学生存在相同课堂表现。
5. 某个小组获得竞赛荣誉。

批量评价时，系统应支持：

1. 多选学生。
2. 按小组选择。
3. 按班级选择。
4. 提交前预览名单。
5. 对个别学生单独调整分值。
6. 批量负向评价时要求填写说明。


## 4. 审核学生自评

学生提交的课堂表现需要教师审核。

教师审核时主要判断：

1. 是否真实发生。
2. 是否属于所选评价入口。
3. 是否选择了正确评价维度。
4. 分值是否合理。
5. 原因卡片是否匹配。
6. 是否重复提交。
7. 是否需要证明材料。

审核结果有三种：

| 操作   | 结果          |
| ---- | ----------- |
| 通过   | 计入学生成长档案和画像 |
| 退回修改 | 学生修改后可重新提交  |
| 不通过  | 不计入画像       |


## 5. 审核学生荣誉

学生提交的荣誉记录也需要教师审核。

教师需要检查：

1. 活动是否真实。
2. 证书图片是否清晰。
3. 五育分类是否正确。
4. 荣誉级别是否准确。
5. 奖项类型是否合理。
6. 是否存在重复提交。

审核通过后，荣誉进入学生成长档案，并同步到画像中的重点荣誉区域。


## 6. 学科成绩导入

教师可以导入本班或任教学科的成绩。

导入流程：

```text
选择年级、班级、学科
→ 选择考试类型
→ 下载 Excel 模板
→ 填写成绩
→ 上传 Excel
→ 系统校验
→ 查看导入预览
→ 确认导入
→ 成绩进入学生档案
→ 更新智育画像
```

教师可以导入的考试类型包括：

1. 单元测试
2. 期中考试
3. 期末考试

系统需要校验：

1. 学生是否存在。
2. 学号和姓名是否匹配。
3. 班级是否正确。
4. 得分是否为空。
5. 得分是否超过满分。
6. 考试类型是否正确。
7. 是否重复导入。


## 7. 查看学生评价记录

教师可以查看学生的历史评价记录，并按条件筛选。

筛选条件包括：

1. 时间
2. 学科或主题
3. 评价入口
4. 评价维度
5. 正向或负向
6. 分值范围
7. 提交来源：教师评价 / 学生自评

该功能方便教师了解学生长期表现，而不是只看某一次评价。


## 8. 查看学生画像

教师可以查看学生个人画像，包括：

1. 五育雷达图
2. 综合成长指数
3. 本学期成长指数
4. 历史成长趋势
5. 学科成绩趋势
6. 近期课堂表现
7. 重点荣誉
8. 优势标签
9. 待提升标签
10. 成长建议

教师也可以查看班级整体情况，例如：

1. 班级五育整体分布。
2. 班级优势领域。
3. 班级薄弱领域。
4. 需要重点关注的学生。
5. 近期负向评价较多的学生。


## 9. 教师端功能边界

教师可以：

1. 评价学生。
2. 审核学生提交内容。
3. 导入成绩。
4. 录入荣誉。
5. 查看学生画像。
6. 查看班级评价情况。

教师不应随意修改系统规则，例如：

1. 五育权重。
2. 全校评价入口。
3. 原因卡片基础库。
4. 荣誉级别。
5. 奖项类型。

这些应由教务管理员统一维护。


# 四、教务管理员角色功能分析

教务管理员主要负责系统的**规则配置、基础数据维护、质量管控和全校数据管理**。

教务管理员不是日常评价的主要执行者，而是保障系统规范运行的角色。


## 1. 管理评价规则

教务管理员可以维护评价体系，包括：

1. 评价入口
2. 评价维度
3. 五育分类
4. 分值规则
5. 正向分值
6. 负向分值
7. 分值说明
8. 推荐使用场景

例如维护：

| 类型   |  分值 |
| ---- | --: |
| 表现良好 |  +1 |
| 表现较好 |  +2 |
| 表现优秀 |  +3 |
| 表现突出 |  +5 |
| 特殊表扬 | +10 |
| 轻微提醒 |  -1 |
| 需要改进 |  -2 |
| 明显不足 |  -3 |
| 严重问题 |  -5 |


## 2. 管理原因卡片

教务管理员可以统一维护原因卡片库。

操作包括：

1. 新增卡片。
2. 修改卡片。
3. 停用卡片。
4. 设置正向或负向。
5. 设置所属评价维度。
6. 设置适用入口。
7. 设置默认推荐分值。

例如：

```text
原因卡片：主动举手发言
评价维度：学习态度
类型：正向
推荐分值：+1 或 +2
适用入口：语文、数学、英语、科学
```

这样可以保证全校评价口径统一。


## 3. 管理五育权重

教务管理员可以配置综合成长指数的计算权重。

默认权重为：

| 五育 |  权重 |
| -- | --: |
| 德育 | 20% |
| 智育 | 35% |
| 体育 | 15% |
| 美育 | 15% |
| 劳育 | 15% |

系统必须校验：

```text
五育权重总和 = 100%
```

如果学校后续调整育人重点，可以由教务管理员统一修改权重。


## 4. 管理荣誉规则

教务管理员可以维护竞赛和荣誉相关基础数据，包括：

1. 活动类型。
2. 五育分类。
3. 荣誉级别。
4. 奖项类型。
5. 是否需要证书图片。
6. 是否允许学生提交。
7. 是否需要教师审核。

荣誉级别包括：

1. 班级
2. 年级
3. 校级
4. 区级
5. 市级
6. 省级
7. 国家级

奖项类型包括：

1. 特等奖
2. 一等奖
3. 二等奖
4. 三等奖
5. 优秀奖
6. 参与奖
7. 优秀个人
8. 优秀小组
9. 荣誉称号
10. 其他


## 5. 管理成绩导入规则

教务管理员可以维护成绩管理相关规则，例如：

1. 学科列表。
2. 考试类型。
3. Excel 导入模板。
4. 成绩字段要求。
5. 满分设置。
6. 导入校验规则。
7. 重复导入处理方式。

核心学科包括：

1. 语文
2. 数学
3. 英语
4. 科学

考试类型包括：

1. 单元测试
2. 期中考试
3. 期末考试


## 6. 基础数据管理

虽然原需求中没有展开，但从系统落地角度看，教务管理员还应负责基础数据管理。

建议包括：

1. 年级管理。
2. 班级管理。
3. 学生信息管理。
4. 教师信息管理。
5. 任课关系管理。
6. 班主任关系管理。
7. 学期管理。
8. 学年管理。

这些数据会影响评价、成绩导入、画像生成等核心功能。


## 7. 数据质量与异常处理

教务管理员应能查看全校数据异常，例如：

1. 成绩导入失败记录。
2. 重复荣誉记录。
3. 异常高分或异常扣分记录。
4. 未审核记录积压情况。
5. 评价入口配置错误。
6. 五育权重配置异常。
7. 学生缺少基础信息。

这类功能有利于保证成长档案数据真实、连续、规范。


## 8. 教务管理员功能边界

教务管理员可以：

1. 配置规则。
2. 维护基础数据。
3. 管理成绩模板。
4. 管理五育权重。
5. 查看全校统计数据。
6. 处理系统级异常数据。

教务管理员不应替代教师完成大量日常评价。
日常评价应主要由任课教师、班主任完成。

# 六、三类角色之间的业务关系

## 1. 学生与教师

学生主要负责提交自己的成长表现，教师负责审核。

```text
学生提交课堂表现
→ 教师审核
→ 审核通过
→ 进入成长档案
→ 更新学生画像
```

```text
学生提交荣誉
→ 教师审核
→ 审核通过
→ 荣誉进入档案
→ 画像重点荣誉更新
```


## 2. 教师与教务管理员

教师负责日常使用，教务管理员负责规则维护。

```text
教务管理员配置评价规则
→ 教师按统一规则评价学生
→ 学生数据进入成长档案
→ 系统生成画像
```

```text
教务管理员维护成绩模板
→ 教师下载模板
→ 教师导入成绩
→ 系统校验并更新智育画像
```


## 3. 学生与教务管理员

学生一般不直接与教务管理员发生业务交互。
教务管理员主要通过规则和基础数据间接影响学生端展示结果。

例如：

```text
教务管理员调整五育权重
→ 系统重新计算成长指数
→ 学生端看到新的成长指数
```


# 七、建议的菜单结构

## 1. 学生端菜单

```text
学生端
├─ 成长画像（综合数据、各学科成绩、雷达图）
├─ 课堂表现（提交课堂表现、审核状态）
├─ 荣誉奖状（提交荣誉奖状、审核状态）
└─ 成长趋势（各个学年的能力成长图）
```


## 2. 教师端菜单

```text
教师端
├─ 快速评价
├─ 审核中心
│  ├─ 学生自评审核
│  └─ 学生荣誉审核
├─ 学生评价记录
├─ 成绩管理
└─ 学生画像（班级列表 => 学生列表 => 学生画像）
```



## 3. 教务管理员端菜单

```text
教务管理端
├─ 基础数据管理
│  ├─ 年级管理
│  ├─ 班级管理
│  ├─ 学生管理
│  ├─ 教师管理
│  └─ 任课关系管理
```


# 学生成长档案画像系统需求与功能逻辑说明

## 一、系统总体目标

学生成长档案画像系统围绕“五育并举”的育人目标，记录学生在德、智、体、美、劳方面的过程性表现、阶段性成绩、竞赛荣誉和综合成长变化。

系统重点解决以下问题：

1. 教师可以快速记录学生在课堂、活动、竞赛中的表现。
2. 学生可以主动提交自己的成长表现，由教师审核后计入档案。
3. 学科成绩可以通过 Excel 批量导入，形成阶段性学习记录。
4. 各类竞赛、评比、荣誉可以统一沉淀到学生成长档案中。
5. 系统根据过程评价、学业成绩、竞赛荣誉等数据，生成学生五育成长画像。
6. 学校、教师、学生、家长可以通过画像了解学生从一年级至今的成长轨迹。


# 二、五育主题与评价范围

系统中的评价主题先按学校实际使用场景设置为八个入口：

1. 科学
2. 英语
3. 数学
4. 语文
5. 礼仪
6. 运动
7. 艺术
8. 劳动

这八个入口再归入五育体系：

| 评价入口 | 所属五育 |
| ---- | ---- |
| 科学   | 智育   |
| 英语   | 智育   |
| 数学   | 智育   |
| 语文   | 智育   |
| 礼仪   | 德育   |
| 运动   | 体育   |
| 艺术   | 美育   |
| 劳动   | 劳育   |

系统使用时，教师或学生先选择评价入口，再选择具体评价维度。


# 三、课堂动态评价系统

## 1. 功能定位

课堂动态评价系统用于快速记录学生在课堂学习、课堂常规、学习成果、竞赛活动中的表现。

该系统不仅支持教师评价学生，也支持学生主动提交自我表现记录，由教师审核后生效。


## 2. 使用角色

### 教师

教师可以：

1. 选择一个学生进行评价。
2. 选择多个学生进行批量评价。
3. 对学生提交的自评记录进行审核。
4. 修改或驳回学生提交的记录。
5. 查看学生历史评价记录。
6. 查看班级整体评价情况。

### 学生

学生可以：

1. 提交自己的课堂表现。
2. 选择评价主题和评价维度。
3. 选择分值和原因卡片。
4. 上传补充说明或证明材料。
5. 查看自己的提交状态。
6. 查看教师审核结果。


## 3. 评价操作流程

### 教师评价流程

```text
教师登录系统
→ 进入课堂动态评价
→ 选择班级
→ 单选或多选学生
→ 选择评价入口：科学、英语、数学、语文、礼仪、运动、艺术、劳动
→ 选择评价维度：学习态度、课堂常规、学业水平、竞赛活动
→ 选择分值
→ 选择评价原因卡片
→ 可填写补充说明
→ 提交评价
→ 评价立即计入学生成长档案
```

### 学生自评提交流程

```text
学生登录系统
→ 进入我的成长表现提交
→ 选择评价入口
→ 选择评价维度
→ 选择分值
→ 选择原因卡片
→ 提交给教师审核
→ 教师审核通过后计入成长档案
→ 审核不通过则退回或作废
```


## 4. 评价入口选择逻辑

第一步先选择评价入口：

| 入口 | 适用场景                 |
| -- | -------------------- |
| 科学 | 科学课堂、实验、探究活动、科技竞赛    |
| 英语 | 英语课堂、口语表达、阅读、听写、英语活动 |
| 数学 | 数学课堂、计算、思维训练、数学竞赛    |
| 语文 | 语文课堂、阅读、写作、朗读、表达     |
| 礼仪 | 文明礼貌、行为规范、班级常规、集体秩序  |
| 运动 | 体育课堂、运动会、体能训练、运动习惯   |
| 艺术 | 音乐、美术、书法、表演、艺术活动     |
| 劳动 | 值日、劳动课、实践活动、家务劳动     |

选择入口后，系统再进入评价维度选择。


## 5. 评价维度选择逻辑

第二步选择评价维度：

1. 学习态度
2. 课堂常规
3. 学业水平
4. 竞赛活动

这四个维度适用于所有评价入口。


## 6. 分值规则

系统分值分为正向加分和负向扣分。

### 正向分值

| 类型   |  分值 | 说明             |
| ---- | --: | -------------- |
| 表现良好 |  +1 | 一般性正向表现        |
| 表现较好 |  +2 | 比较明显的积极表现      |
| 表现优秀 |  +3 | 明显优于日常表现       |
| 表现突出 |  +5 | 值得重点表扬的表现      |
| 特殊表扬 | +10 | 重大进步、突出贡献、重要荣誉 |

### 负向分值

| 类型   | 分值 | 说明               |
| ---- | -: | ---------------- |
| 轻微提醒 | -1 | 轻微问题，需要提醒        |
| 需要改进 | -2 | 问题较明显，需要关注       |
| 明显不足 | -3 | 多次出现或影响较大        |
| 严重问题 | -5 | 重大违纪或严重影响课堂、活动秩序 |


## 7. 分值使用建议

为了避免评价过度随意，系统可以提供推荐规则：

### +1 适用场景

* 当堂表现比平时更积极。
* 完成了一项基本任务。
* 有一次值得肯定的小进步。

### +2 适用场景

* 课堂持续认真。
* 主动参与学习活动。
* 能较好完成教师布置的任务。

### +3 适用场景

* 在课堂中表现稳定优秀。
* 回答、作业、合作表现明显突出。
* 能对同学产生正向影响。

### +5 适用场景

* 在班级活动中表现突出。
* 帮助小组完成重要任务。
* 学习成果明显优秀。

### +10 适用场景

* 获得重要荣誉。
* 在大型活动中作出突出贡献。
* 有非常明显的成长突破。

### -1 适用场景

* 偶发性不专注。
* 偶尔未按要求完成。
* 经提醒后能改正。

### -2 适用场景

* 问题较明显。
* 需要教师多次提醒。
* 对个人学习产生影响。

### -3 适用场景

* 多次出现同类问题。
* 对他人或课堂秩序造成影响。
* 学习任务完成质量明显不足。

### -5 适用场景

* 严重违反课堂规则。
* 明显影响他人学习。
* 在活动或竞赛中出现严重失误、违纪或不当行为。


# 四、评价原因卡片设计

评价原因卡片用于帮助教师和学生快速选择评价原因，减少手动输入，提高评价效率。

每个评价维度分为：

1. 正向原因
2. 负面原因

每类至少 10 个选项。


## 1. 学习态度原因卡片

### 正向原因

1. 主动举手发言
2. 认真听讲
3. 积极参与讨论
4. 主动完成学习任务
5. 课前准备充分
6. 学习目标明确
7. 遇到问题主动请教
8. 能坚持完成较难任务
9. 学习过程中专注投入
10. 能主动帮助同学学习
11. 对新知识表现出浓厚兴趣
12. 能按时完成课堂练习

### 负面原因

1. 注意力不集中
2. 学习准备不足
3. 作业完成不认真
4. 课堂参与度低
5. 不愿主动思考
6. 遇到困难容易放弃
7. 上课走神
8. 未按要求完成学习任务
9. 学习用品准备不齐
10. 对课堂活动缺乏兴趣
11. 需要多次提醒才能进入学习状态
12. 自主学习意识较弱


## 2. 课堂常规原因卡片

### 正向原因

1. 遵守课堂纪律
2. 认真记录笔记
3. 坐姿端正
4. 积极配合教师

6. 发言前主动举手
7. 能安静倾听他人发言

10. 能维护良好课堂秩序
11. 学习用品摆放整齐

### 负面原因

1. 随意讲话
2. 上课迟到
3. 未按要求准备学习用品
4. 影响他人学习
5. 未经允许离开座位
6. 课堂活动中不遵守规则
7. 插话或打断他人发言
8. 不认真倾听教师讲解
9. 小组合作中影响秩序
10. 课堂准备拖拉
11. 学习用品杂乱影响学习
12. 经提醒后仍未改正

# 五、课堂动态评价审核逻辑

## 1. 教师直接评价

教师提交的评价默认有效，直接进入学生成长档案。

适用场景：

* 课堂即时评价
* 班级日常评价
* 活动现场评价
* 教师课后补录评价


## 2. 学生自评提交

学生提交的评价不直接生效，需要教师审核。

学生提交后，记录状态为：

```text
待审核
```

教师审核后有三种处理方式：

### 审核通过

评价计入学生成长档案。

```text
待审核 → 已通过 → 计入画像
```

### 退回修改

教师认为内容基本合理，但需要学生补充说明或调整分值。

```text
待审核 → 退回修改 → 学生重新提交
```

### 审核不通过

教师认为内容不真实、不符合要求或重复提交。

```text
待审核 → 不通过 → 不计入画像
```


## 3. 学生自评审核重点

教师审核时重点看：

1. 是否真实发生。
2. 是否属于该评价入口。
3. 是否选择了正确的评价维度。
4. 分值是否合理。
5. 原因卡片是否匹配。
6. 是否存在重复提交。
7. 是否需要补充说明或证明材料。


## 4. 批量评价逻辑

教师可以多选学生后进行批量评价。

适用场景：

1. 小组合作表现优秀。
2. 多名学生共同参加活动。
3. 全班完成某项任务。
4. 多名学生存在相同课堂表现。
5. 某个小组获得竞赛荣誉。

批量评价时：

* 同一评价内容会同时应用到所选学生。
* 教师仍可在提交前对个别学生调整分值。
* 系统应提示教师避免误选学生。
* 批量负向评价建议要求填写补充说明。


# 六、学科成绩管理系统

## 1. 功能定位

学科成绩管理系统用于记录学生在各学科的阶段性学习结果，是学生智育画像的重要来源。

系统支持导入：

1. 单元测试
2. 期中考试
3. 期末考试

主要服务对象：

* 任课教师
* 班主任
* 教务管理人员
* 学生
* 家长


## 2. 支持学科

核心学科包括：

1. 语文
2. 数学
3. 英语
4. 科学

这些学科归入智育。

后续也可以扩展其他课程，但当前系统以这四门学科为主。


## 3. 成绩导入流程

```text
教师或教务人员进入成绩管理
→ 选择年级、班级、学科
→ 选择考试类型：单元测试、期中考试、期末考试
→ 下载 Excel 模板
→ 填写学生成绩
→ 上传 Excel 文件
→ 系统自动校验
→ 显示导入预览
→ 确认导入
→ 成绩进入学生档案
→ 更新学生智育画像和成长趋势
```


## 4. Excel 导入要求

导入内容应包括：

1. 学生姓名
2. 学号
3. 班级
4. 学科
5. 考试名称
6. 考试类型
7. 满分
8. 得分
9. 考试时间
10. 任课教师


## 5. 导入校验逻辑

系统需要在导入时进行校验。

### 基础校验

1. 学生是否存在。
2. 学号和姓名是否匹配。
3. 班级是否正确。
4. 学科是否正确。
5. 得分是否为空。
6. 得分是否超过满分。
7. 满分是否有效。
8. 考试类型是否正确。
9. 是否存在重复导入。
10. 文件格式是否正确。

### 异常处理

导入失败时，系统应明确提示：

* 哪一行出错。
* 哪个字段出错。
* 出错原因是什么。
* 如何修改。

例如：

```text
第 12 行：学生姓名与学号不匹配。
第 18 行：数学得分 105 超过满分 100。
第 21 行：考试类型为空。
```


## 6. 成绩展示逻辑

学生画像中应展示：

1. 本学期各学科成绩。
2. 单元测试变化趋势。
3. 期中、期末成绩对比。
4. 单科成长趋势。
5. 四门学科综合表现。
6. 优势学科。
7. 待提升学科。


## 7. 成绩分析逻辑

系统不只展示分数，还应展示成长变化。

### 单科趋势

例如：

```text
数学从第一次单元测试 82 分，提升到第二次单元测试 90 分，系统标记为“数学进步明显”。
```

### 阶段对比

例如：

```text
期中英语 86 分，期末英语 92 分，系统显示英语本学期提升 6 分。
```

### 学科均衡分析

系统可以判断学生是否存在明显偏科。

例如：

```text
语文、英语表现稳定，数学波动较大，科学成绩有提升空间。
```


# 七、日常竞赛和荣誉管理系统

## 1. 功能定位

日常竞赛和荣誉管理系统用于记录学生参加各类活动、竞赛、评比、荣誉的情况。

它是学生五育画像的重要补充，能够体现学生在课堂成绩之外的综合发展。


## 2. 记录内容

每条竞赛或荣誉记录包括：

1. 学生
2. 活动名称
3. 活动类型
4. 所属五育分类
5. 级别
6. 奖项
7. 证书图片
8. 获奖时间


## 3. 活动类型

活动类型建议包括：

1. 学科竞赛
2. 科技创新
3. 阅读活动
4. 书写比赛
5. 英语活动
6. 体育比赛
7. 运动会项目
8. 艺术展演
9. 绘画比赛
10. 音乐比赛
11. 劳动实践
12. 礼仪评比

## 7. 荣誉录入流程

```text

→ 填写活动名称
→ 选择活动类型
→ 选择所属五育分类
→ 选择级别
→ 选择奖项
→ 上传证书图片
→ 填写获奖时间
→ 提交
→ 荣誉进入学生成长档案
→ 学生画像中的重点荣誉同步更新
```


## 8. 学生提交荣誉流程

学生也可以主动提交荣誉信息。

```text
学生进入我的荣誉提交
→ 选择活动名称
→ 选择活动类型
→ 选择所属五育分类
→ 选择级别和奖项
→ 上传证书图片
→ 填写获奖时间
→ 提交教师审核
→ 教师审核通过后进入成长档案
```

学生提交的荣誉必须审核后才生效。


## 9. 重点荣誉展示逻辑

学生画像首页展示重点荣誉时，优先展示：

1. 级别较高的荣誉。
2. 最近获得的荣誉。
3. 与学生优势标签相关的荣誉。
4. 五育分类中有代表性的荣誉。
5. 教师手动标记为重点展示的荣誉。

例如：

```text
校级数学竞赛一等奖
区级运动会跳远二等奖
校园艺术节优秀表演奖
文明礼仪之星
劳动实践优秀个人
```


# 八、学生画像设计

## 1. 功能定位

学生画像是系统的核心展示结果，用于综合呈现学生在五育方面的发展状态。

画像不是简单排名，而是展示学生：

1. 当前发展情况。
2. 本学期成长表现。
3. 从一年级至今的历史变化。
4. 优势领域。
5. 待提升方向。
6. 重要荣誉。
7. 综合成长趋势。


## 2. 学生画像首页内容

学生画像首页包含：

1. 学生基本信息
2. 五育综合雷达图
3. 本学期成长指数
4. 历史成长趋势曲线
5. 重点荣誉展示
6. 近期课堂表现
7. 学科成绩表现
8. 优势标签
9. 待提升标签
10. 成长建议


## 3. 学生基本信息

展示内容包括：

1. 姓名
2. 学号
3. 年级
4. 班级
5. 班主任
6. 当前学期
7. 入学年份
8. 当前成长指数


## 4. 五育综合雷达图

雷达图展示五个维度：

1. 德育
2. 智育
3. 体育
4. 美育
5. 劳育

每个维度展示当前阶段得分或等级。

### 雷达图作用

教师和家长可以直观看到学生是否全面发展。

例如：

* 智育较高，体育较弱：说明学生学业表现较好，但运动参与不足。
* 美育突出，德育稳定：说明学生在艺术方面有明显优势。
* 五育较均衡：说明学生整体发展比较全面。


## 5. 综合成长指数

综合成长指数按照五育权重计算：

| 五育 |  权重 |
| -- | --: |
| 德育 | 20% |
| 智育 | 35% |
| 体育 | 15% |
| 美育 | 15% |
| 劳育 | 15% |

计算逻辑：

```text
综合成长指数 =
德育得分 × 20%
+ 智育得分 × 35%
+ 体育得分 × 15%
+ 美育得分 × 15%
+ 劳育得分 × 15%
```


## 6. 各五育得分来源

### 德育得分来源

主要来自：

1. 礼仪评价
2. 课堂常规评价
3. 文明礼仪荣誉
4. 班级服务表现
5. 德育类活动记录

### 智育得分来源

主要来自：

1. 语文课堂评价
2. 数学课堂评价
3. 英语课堂评价
4. 科学课堂评价
5. 单元测试成绩
6. 期中考试成绩
7. 期末考试成绩
8. 学科竞赛荣誉

### 体育得分来源

主要来自：

1. 运动课堂表现
2. 体育活动参与
3. 体育比赛成绩
4. 运动会荣誉
5. 日常运动表现

### 美育得分来源

主要来自：

1. 艺术课堂表现
2. 音乐、美术活动记录
3. 艺术比赛荣誉
4. 展演活动参与
5. 审美表达表现

### 劳育得分来源

主要来自：

1. 劳动课堂表现
2. 值日值岗表现
3. 劳动实践活动
4. 劳动技能比赛
5. 家校劳动反馈


## 7. 本学期成长指数

本学期成长指数用于展示学生当前学期的发展情况。

它反映的是学生在本学期内的综合成长表现，主要依据：

1. 本学期课堂动态评价。
2. 本学期单元、期中、期末成绩。
3. 本学期竞赛和荣誉。
4. 本学期活动参与情况。

### 展示方式

可以显示：

```text
本学期成长指数：86
较上学期提升：+4
当前优势：数学、礼仪、艺术
需要关注：运动参与、课堂专注度
```


## 8. 历史成长趋势曲线

历史成长趋势曲线从一年级开始展示，直到当前年级。

### 展示维度

可以展示：

1. 综合成长指数趋势
2. 德育趋势
3. 智育趋势
4. 体育趋势
5. 美育趋势
6. 劳育趋势

### 展示周期

可以按以下周期查看：

1. 按学期查看
2. 按学年查看
3. 从一年级至今查看

### 示例

```text
一年级上：76
一年级下：78
二年级上：81
二年级下：83
三年级上：86
```

系统通过曲线展示学生的长期成长，而不是只看某一次考试或某一天表现。


## 9. 重点荣誉展示

学生画像首页应展示重点荣誉。

### 展示内容

1. 荣誉名称
2. 活动名称
3. 所属五育
4. 荣誉级别
5. 奖项
6. 获奖时间
7. 证书图片

### 展示规则

优先展示：

1. 高级别荣誉。
2. 最近获得的荣誉。
3. 五育代表性荣誉。
4. 教师标记的重要荣誉。
5. 能体现学生优势的荣誉。


# 九、学生画像生成逻辑

## 1. 数据进入画像的路径

系统画像主要由三类数据生成：

```text
课堂动态评价
+ 学科成绩
+ 日常竞赛和荣誉
= 学生成长画像
```


## 2. 课堂评价进入画像

教师直接评价后，立即进入画像统计。

学生自评通过审核后，进入画像统计。

未审核、审核不通过、已撤回的记录，不进入画像统计。


## 3. 学科成绩进入画像

成绩导入成功后，系统自动更新智育画像。

单元测试影响阶段性趋势。

期中考试和期末考试影响本学期智育表现。


## 4. 荣誉进入画像

荣誉录入成功后，进入对应五育画像。

例如：

* 数学竞赛一等奖 → 智育
* 文明礼仪之星 → 德育
* 校运会跳远二等奖 → 体育
* 绘画比赛一等奖 → 美育
* 劳动实践优秀个人 → 劳育


## 5. 标签生成逻辑

系统根据学生表现自动生成标签。

### 正向标签示例

1. 学习态度积极
2. 课堂表现稳定
3. 数学思维突出
4. 语文表达优秀
5. 英语参与度高
6. 科学探究能力强
7. 礼仪表现优秀
8. 运动能力突出
9. 艺术表现活跃
10. 劳动习惯良好

### 待提升标签示例

1. 课堂专注度需提升
2. 作业质量需关注
3. 学习准备不够充分
4. 课堂常规需加强
5. 数学基础需巩固
6. 语文阅读理解需提升
7. 英语表达需加强
8. 运动参与不足
9. 艺术活动参与较少
10. 劳动主动性需提升

# 十四、系统核心闭环

整个系统的核心闭环如下：

```text
教师评价 / 学生自评
→ 教师审核
→ 进入成长档案
→ 汇总五育表现
→ 生成学生画像
→ 展示成长趋势
→ 形成成长建议
→ 反向指导教学和学生发展
```

系统的核心价值不是简单打分，而是通过长期、连续、真实的数据，帮助教师和家长看见学生成长过程，帮助学生认识自己的优势和改进方向。
